# E2EE Threat Model

Версия: 0.1

## 1. Допущения (Assumptions)

- **Сервер — полудоверенный (honest-but-curious):** не подменяет шифротекст, но может пытаться его читать.
- **Сеть:** возможна MITM-атака (подмена prekey bundle, downgrade).
- **Клиент:** ключи генерируются на клиенте; сервер хранит только публичные данные (identity_key, signed_prekey, подпись).

## 2. Угрозы

### 2.1 Пассивный перехват

| Угроза | Защита |
|--------|--------|
| Перехват plaintext | E2EE — сервер видит только шифротекст |
| Анализ трафика | TLS (HTTPS/WSS), шифрование на транспортном слое |

### 2.2 Активный MITM (подмена prekey bundle)

| Угроза | Описание |
|--------|----------|
| Подмена prekey bundle | Атакующий перехватывает запрос `GET /keys/{address}`, подменяет identity_key и signed_prekey на свои. Клиент шифрует для атакующего — тот расшифровывает. |

**Митигация:**

- **Safety number / fingerprint:** пользователи сравнивают отпечаток (SHA-256 от пары identity public keys) лично. Если совпадает — канал не скомпрометирован.
- **Signal Protocol:** X3DH + Double Ratchet дают forward secrecy; компрометация ключей не раскрывает старые сообщения.

### 2.3 Компрометация ключей

| Угроза | MVP | Signal |
|--------|-----|--------|
| Кража identity/signed_prekey | Все сообщения расшифровываются | Double Ratchet: старые сообщения защищены |
| Кража серверной БД | Только публичные ключи; приватные только на клиенте | Аналогично |

### 2.4 Сервер

| Угроза | Защита |
|--------|--------|
| Подмена шифротекста | Клиент проверяет MAC/authenticated encryption; при неверной подписи — decrypt failed |
| Replay | Timestamp в транзакциях, порядок событий |

## 3. Аутентификация шифротекста

- **MVP (NaCl secretbox):** Poly1305 MAC — подделка ciphertext приводит к decrypt failed.
- **Signal (sig1:):** HMAC в Double Ratchet — аналогично.

Сервер не может подменить ciphertext так, чтобы клиент принял поддельное сообщение.

## 4. Рекомендации

1. **Проверка Safety number** — для конфиденциальных каналов сравнивать fingerprint при личной встрече.
2. **Signal по умолчанию** — использовать Signal Protocol (persistent store, IndexedDB) для forward secrecy.
3. **Бэкап ключей** — зашифрованный бэкап с паролем; без пароля ключи не восстанавливаются.
