# E2EE-бэкап с зависимостью от сервера

Версия: **0.1**

## Цель

Пользователь хранит зашифрованный бэкап ключей на своём устройстве. Без активного сервера данные нельзя расшифровать.

## Модель

- **Бэкап** = зашифрованный blob (identity secret, signed prekey secret)
- **Ключ K** = KDF(пароль_пользователя, salt_сервера)
- **Сервер хранит** salt (уникальный на user+device)
- **Пользователь хранит** encrypted_blob (локально или в своём облаке)

Расшифровка: пароль + salt от сервера (после аутентификации) → K → decrypt.

## Поток

### Создание бэкапа

1. Пользователь вводит пароль (дополнительный к паролю входа или тот же)
2. Клиент вызывает `POST /api/v1/backup/prepare` → сервер создаёт/возвращает salt
3. K = PBKDF2(пароль, salt, 100000)
4. Клиент шифрует keys (identitySecret, signedPrekey.secret) с K
5. Пользователь сохраняет blob (скачивает файл)

### Восстановление

1. Пользователь загружает blob, вводит пароль, логинится
2. Клиент вызывает `GET /api/v1/backup/salt` → сервер возвращает salt
3. K = PBKDF2(пароль, salt, 100000)
4. Расшифровывает blob → восстанавливает keys в localStorage

## API

- `POST /api/v1/backup/prepare` — подготовка (создаёт salt, возвращает base64)
- `GET /api/v1/backup/salt` — получить salt для восстановления

## Формат blob

```
version(1) || salt_id(16) || nonce(24) || ciphertext
```

- version = 1
- salt_id = идентификатор (для проверки соответствия salt)
- nonce = NaCl secretbox nonce
- ciphertext = secretbox(JSON(keys), nonce, K)
