# Архитектура — хаб и самохостинг

## Роли

### Основной домен (хаб)

Один главный инстанс, доступный по основному домену. Пользователи, которые не разворачивают свой сервер, входят через него.

- **Точка входа** для большинства пользователей
- **HA**: несколько реплик server за балансировщиком, PostgreSQL (с опциональной репликой)
- **Сеть**: федерация с самохостинг-инстансами

### Самохостинг

Пользователи разворачивают свой инстанс и подключают его к федерации.

- **Расширение сети**: больше ёмкости, географическое распределение
- **Изоляция**: свои данные, свой контроль
- **Федерация**: обмен сообщениями с хабом и другими инстансами

## Схема

```
                    [main-domain.com] — основной хаб
                              │
              ┌───────────────┼───────────────┐
              │               │               │
         [Traefik]      [server 1]      [server 2]
              │               │               │
              └───────────────┼───────────────┘
                              │
                    [PostgreSQL] [Redis]
                              │
              ────────────────┼─────────────── федерация
                              │
         [self-host A]   [self-host B]   [self-host C]
```

## Отказоустойчивость

- **Один сервер упал** — Traefik направляет трафик на оставшиеся
- **Все серверы упали** — нужен перезапуск
- **Новый инстанс после полного падения** — система снова работает, данные потеряны (нужен бэкап)
- **PostgreSQL replica** (опционально) — чтение с реплики, отказ primary = failover
