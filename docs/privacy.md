# Privacy-by-Design

Принципы минимизации метаданных и логов для инстанса мессенджера.

---

## 1. По умолчанию (default-on)

### 1.1 Логирование

| Данные | Действие по умолчанию |
|--------|------------------------|
| IP-адреса пользователей | **Не логировать** в application logs |
| User ID в логах | **Не включать** в сообщения об ошибках |
| Содержимое сообщений | **Не хранится** (только шифртекст) |
| Время последней активности | Опционально, отключаемо |

### 1.2 Reverse proxy / Load balancer

- Не передавать `X-Forwarded-For` в backend, если не требуется для rate limit по IP
- Либо анонимизировать (например, обнулять последний октет)
- Access logs: минимизировать или отключать по умолчанию

### 1.3 Очередь сообщений

- Хранить только: `event_id`, `sender`, `recipient`, `timestamp`, `ciphertext`, `session_id`
- **TTL**: удаление после доставки или по истечении срока (например, 7 дней)
- Не хранить: IP отправителя, User-Agent, геолокацию

### 1.4 Read receipts / Typing indicators

- **По умолчанию выключены** — увеличивают объём метаданных (кто когда прочитал, кто печатает)
- В MVP не реализуем; при добавлении — opt-in на стороне пользователя

---

## 2. Метрики

Только агрегированные показатели:

- Количество активных пользователей (без привязки к user_id)
- Количество сообщений в единицу времени (агрегат)
- Ошибки по типам (без user_id в лейблах)
- Латентность API (перцентили)

**Запрещено в метриках:**

- `user_id`, `device_id`, IP в лейблах
- Индивидуальные трассировки с идентификаторами пользователей

---

## 3. Media (файлы)

- Content-URI = content hash → нельзя определить структуру/имена файлов по URI
- Сервер не генерирует превью, не анализирует содержимое
- Превью — только на клиенте после расшифровки

---

## 4. Переключатели для админа (opt-in)

Админ инстанса может **включить** для целей модерации/антиспама/форензики:

| Опция | Описание | Рекомендуемое хранение |
|-------|----------|------------------------|
| `log_client_ips` | Логировать IP при подключении | Отдельная таблица, короткий TTL |
| `log_failed_auth` | Логировать неудачные попытки входа | Без user_id, только IP + timestamp |
| `retention_days` | Срок хранения недописанных сообщений | По умолчанию 7 |
| `analytics_extended` | Расширенные метрики | По необходимости |

Эти опции **выключены по умолчанию** и должны явно настраиваться в конфиге.

---

## 5. Регистрация и идентификация

- Регистрация: **никнейм** + ключи, без email/телефона
- User ID = `@localpart:domain` — привязка только к домену инстанса
- Нет требования верификации identity (по дизайну)

---

## 6. Federation

- S2S: передаются только подписанные транзакции с ciphertext
- Исходный IP отправителя не передаётся в транзакции
- Каждый сервер видит только: sender domain, recipient (свой), ciphertext, timestamp

---

## 7. Резюме

| Принцип | Реализация |
|---------|------------|
| Минимизация данных | Собирать только необходимое для работы |
| Отсутствие plaintext на сервере | E2EE, сервер хранит только шифртекст |
| Анонимизация логов | Нет IP, user_id в логах по умолчанию |
| TTL и ротация | Очереди, логи — ограниченный срок жизни |
| Opt-in для расширенного сбора | Админ явно включает при необходимости |
