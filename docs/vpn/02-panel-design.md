# VPN Panel — панель управления по аналогии с 3X-UI / Marzban

## Идея

Сделать для мессенджера **панель управления VPN** — не сам VPN, а админка для самохостинга: пользователи, лимиты, конфиги, статистика, при желании — несколько нод. Логика как у 3X-UI/Marzban, но завязанная на наш homeserver и аккаунты мессенджера.

---

## 3X-UI vs Marzban — кратко

| | 3X-UI | Marzban |
|---|-------|---------|
| Назначение | Простая панель для одного сервера | Система для сотен аккаунтов, несколько нод |
| Протоколы | Xray (VMess, VLESS, Trojan, SS, WireGuard...) | Xray-core |
| API | Ограниченный | Полноценный REST API |
| Масштаб | Семья / небольшая команда | Много пользователей, биллинг, интеграции |

---

## Что делаем мы

**Вариант A: Лёгкая панель (как 3X-UI)**

- Один homeserver = один VPN-сервер
- Пользователи = аккаунты мессенджера
- Web-UI: список протоколов, скачать конфиг, лимиты (трафик/срок)
- Админ: включить/выключить VPN, квоты, просмотр статистики

**Вариант B: Системная панель (как Marzban)**

- Несколько нод (несколько VPN-серверов)
- REST API для интеграций (биллинг, бот, выдача доступов)
- Роли: админ, оператор, пользователь
- Централизованное управление всеми нодами

**Рекомендация**: начать с варианта A, потом развить до B.

## Протоколы: WireGuard/OpenVPN vs Xray

**Сейчас**: WireGuard, OpenVPN (stub). Простые, но WireGuard блокируют чаще.

**3X-UI/Marzban**: Xray-core — VMess, VLESS, Trojan, Shadowsocks, маскировка под HTTPS, работает в жёстких сетях.

**Имеет смысл добавить**:

1. **Xray** (VMess/VLESS/Trojan) — для обхода блокировок, один из приоритетов.
2. **WireGuard** — оставить для скоростных и «чистых» сетей.
3. **OpenVPN TCP 443** — как запасной вариант.

Панель тогда будет выбирать: «WireGuard (быстро)» / «Xray (обход блокировок)».

## Функции панели (поэтапно)

### Реализовано
- [x] Самообслуживание: скачать конфиг, отозвать свой конфиг
- [x] Каждый пользователь видит только свои конфиги
- [x] Multi-node: выбор ноды при скачивании
- [x] HA: состояние в PostgreSQL, работает с несколькими репликами
- [x] **Xray**: VMess, VLESS, Trojan — генерация URL (vmess://, vless://, trojan://), обход DPI

---

## Безопасность

1. **Панель** — только по HTTPS, доступ по паролю (JWT).
2. **Изоляция** — пользователь видит только свои конфиги.
3. **Разделение ролей** — админ vs обычный пользователь.
4. **Регулярные обновления** — Xray/WireGuard, панель, ОС.
5. **Фаервол** — только нужные порты (51820, 443 и т.п.).
6. **Логи** — базовый мониторинг доступа и аномалий.
