## Проблема: не везде работает один протокол

| Протокол | Где работает | Где блокируется |
|----------|--------------|-----------------|
| **WireGuard** (UDP 51820) | Дома, офис, большинство мобильных сетей | Корпоративные файрволы (блокируют UDP), Китай, некоторые провайдеры |
| **OpenVPN TCP 443** | Почти везде (маскируется под HTTPS) | Редко |
| **OpenVPN UDP** | Быстрее TCP, но UDP могут резать | Как WireGuard |

## Решение: несколько протоколов, один клик на переключение

```
┌─────────────────────────────────────────────────────────────┐
│  Веб-клиент: "VPN"                                          │
├─────────────────────────────────────────────────────────────┤
│  WireGuard     [Скачать]  ← быстро, рекомендуется           │
│  OpenVPN 443   [Скачать]  ← если WireGuard не подключается  │
└─────────────────────────────────────────────────────────────┘
```

**UX**: пользователь скачивает WireGuard. Не работает? Открывает VPN снова, жмёт "OpenVPN 443" — получает новый конфиг. Тот же аккаунт, тот же сервер, другой протокол.

## Архитектура

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  VPN Control    │     │  Protocol        │     │  VPN Data       │
│  Plane (API)    │────►│  Adapters        │────►│  Plane          │
│                 │     │  (WG, OpenVPN)   │     │  (containers)   │
└─────────────────┘     └──────────────────┘     └─────────────────┘
        │                         │
        │                         │ Генерирует конфиг
        │                         │ Регистрирует peer
        ▼                         ▼
┌─────────────────┐     ┌──────────────────┐
│  Auth (JWT)     │     │  vpn_peers       │
│  device_id      │     │  (БД)            │
└─────────────────┘     └──────────────────┘
```

## API

```
GET  /api/v1/vpn/protocols
     → { "protocols": [
           { "id": "wireguard", "name": "WireGuard", "hint": "Быстро, рекомендуется" },
           { "id": "openvpn-tcp443", "name": "OpenVPN (TCP 443)", "hint": "Если WireGuard заблокирован" }
        ] }

GET  /api/v1/vpn/config/{protocol}?format=file
     Authorization: Bearer <token>
     → 200: application/x-config (WireGuard .conf или OpenVPN .ovpn)
     → или JSON с полем config для вставки
```

## Протоколы (MVP)

### 1. WireGuard
- Генерация ключевой пары клиента
- Конфиг с endpoint сервера (из env)
- Peer добавляется в wg через shared volume + sidecar или `wg set`

### 2. OpenVPN TCP 443
- Fallback для ограничивающих сетей
- Генерация .ovpn (сертификат или статический ключ)
- Endpoint: `tcp://vpn.example.org:443`

## Конфигурация (env)

```
VPN_ENABLED=true
VPN_WIREGUARD_ENDPOINT=wg.example.org:51820
VPN_WIREGUARD_SERVER_PUBLIC_KEY=base64...
VPN_OPENVPN_ENDPOINT=vpn.example.org:443
# и т.д.
```

## Безопасность

- Конфиг привязан к `device_id` (как мессенджер)
- Один конфиг на устройство на протокол (перезапрос = ротация ключа)
- Админ может отключить протоколы через env
- Ревизация: таблица `vpn_peers` хранит pubkey для отзыва
