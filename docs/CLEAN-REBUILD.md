# Полная очистка и пересборка

Документ описывает, как полностью очистить проект и пересобрать его с нуля — без остатков старых данных в БД, файлах и кэшах. Полезно при сбросе на чистую среду, переходе на другой сервер или после компрометации.

## Что удаляется

| Компонент | Содержимое | После очистки |
|-----------|------------|---------------|
| **postgres_data** | БД: пользователи, сообщения, комнаты, ключи, VPN конфиги | Пустая БД, миграции применяются заново |
| **minio_data** | Загруженные файлы (аватары, вложения) | Пустой bucket |
| **redis_data** | Кэш, сессии WebSocket Pub/Sub | Пустой Redis |
| **server_data** | Ключи федерации, при необходимости — прочее | Чистый сервер |
| **traefik_letsencrypt** | Сертификаты Let's Encrypt | Сохраняются (по умолчанию не удаляем) |
| **xray_certs** | Сертификаты VPN | Сохраняются |
| **mesh-ca** (main) | CA для mTLS coordinator | Сохраняются |
| **federation** (selfhost) | mTLS клиентские сертификаты | Сохраняются |

## Скрипт очистки

```bash
./scripts/clean-rebuild.sh [main|selfhost|dev]
```

- **main** — deploy/main (hub с Traefik, HA)
- **selfhost** — deploy/selfhost (одиночный сервер)
- **dev** — deploy/dev (разработка)

Скрипт:
1. Останавливает контейнеры
2. Удаляет volumes
3. Пересобирает образы без кэша
4. Запускает систему заново

## Ручная очистка

Если скрипт не подходит под вашу конфигурацию:

```bash
# Main (с mesh: coordinator и backup-receiver)
docker compose -p main -f deploy/main/docker-compose.yml -f deploy/main/docker-compose.mesh.yml --env-file deploy/main/.env down -v

# Удалить конкретные volumes
docker volume rm main_postgres_data main_minio_data main_redis_data main_server_data

# Пересобрать и запустить
docker compose -p main -f deploy/main/docker-compose.yml -f deploy/main/docker-compose.mesh.yml --env-file deploy/main/.env up -d --build
```

## Очистка на стороне клиента

Для полного сброса в браузере:

1. F12 → Application → Storage → Clear site data  
   или  
2. Удалить данные сайта для вашего домена

Иначе в localStorage останутся старые E2EE-ключи, токены и `lastRead`.

## Безопасность при пересборке

- **Секреты (.env)** не трогаются — JWT_SECRET, пароли БД и т.п. остаются. Для смены секретов отредактируйте `.env` и перезапустите.
- **Новые ключи** — после очистки БД все пользователи должны заново зарегистрироваться; старые аккаунты исчезнут.
- **Миграции** — применяются автоматически при первом запуске server против пустой БД.

## После очистки

1. Проверьте доступность сайта
2. Зарегистрируйте нового пользователя
3. При необходимости настройте Push (VAPID), VPN, федерацию

## Жёсткие требования по безопасности

Для максимальной защиты при чистой пересборке:

- **Новые секреты** — смените JWT_SECRET, POSTGRES_PASSWORD, MINIO_ROOT_PASSWORD, DB_ENCRYPTION_KEY в `.env` перед первым запуском после очистки
- **Генерация**: `openssl rand -base64 32` для каждого секрета
- **БД** — используйте DB_ENCRYPTION_KEY для шифрования чувствительных полей (сервер поддерживает)
- **HTTPS** — в продакшене всегда используйте Let's Encrypt, не HTTP
